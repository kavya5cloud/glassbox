# Plugin SDK

Plugins are how Glassbox stays future-proof.

Plugin categories:

- providers
- frameworks
- IDEs
- browser extensions
- CI integrations
- cloud connectors
- interpreters
- evaluators

Plugin requirements:

- explicit manifest
- capability discovery
- versioned compatibility checks
- sandboxed execution where appropriate
- signed or trusted distribution for marketplace components

Community plugins should remain maintainable by forcing a small, explicit interface and a clear compatibility contract.
